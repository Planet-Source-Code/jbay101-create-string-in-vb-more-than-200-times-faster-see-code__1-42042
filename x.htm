FastString Benchmark

Visual Basic stores it's strings in a type refered to in C++ as a BSTR. This is an ActiveX-type format which is defined in the OLE Automation library. In Visual Basic, when you dynamicly create a string, VB automaticly fills it with data. However, when you deal with large strings, this can take a lot of time, and most of the time you don't want this. So, included in this project is a function which creates the string and doesn't fill it with data. This is a lot faster, as this benchmark shows.
 

The benchmark results are below:

Using test buffer of 30,000 KB.

Function Time (ms) Description
AllocString_Space 333.74 Allocate storage using the Visual Basic 'Space' function.
AllocString_StringASC 395.58 Allocate storage using the Visual Basic 'String' function and passing it a character code.
AllocString_StringCHR 479.57 Allocate storage using the Visual Basic 'String' function and passing it a character.
AllocString_ADVANCED 11.96 Using the API to allocate the storage.
